Description

In this initial stage, you will be laying the foundation for your CLI-based chat assistant. This involves integrating the OpenAI Chat Completions API into a Python program and initiating a basic conversation with your assistant.

Your first task is to initialize the Chat Completions API provided by OpenAI. This API is the backbone of your assistant, enabling it to generate human-like text based on the prompts it receives. You will need to install the OpenAI Python library, import the necessary modules, and use your OpenAI API key to authenticate your program and create a chat completion session. Don't forget to store your key as an environment variable for security.

Important Note:
⚠️ This project involves the use of private keys (OpenAI API keys) and .env files. Please ensure you DO NOT expose your keys by hardcoding them directly into the solution. Instead, always reference them securely from your .env file. Additionally, do not include your .env file in your project submission. Double-check your code for any accidental key exposure before submission. If sharing code snippets, mask sensitive values.

Once you've initialized the API, you're ready to start a conversation with your assistant. Begin with a system message to set the behavior of the assistant. A system message might instruct the assistant about its role or provide a context for the conversation.

Something like this could work to get different responses:

"You are an helpful assistant for a simple CLI chat. Only respond with text messages. Get creative with the answers!"

Next, send a user message asking "What are you?" to the assistant. This is a simple prompt that will get the conversation rolling and allow you to test the assistant's ability to generate a meaningful response.

Finally, output the assistant's response to your console. You can check the example to see the format of the output.

Objectives

In this stage, you should:

  1. Initialize the OpenAI Chat Completions API in a Python program.

  2. Create a chat completion session.

  3. Start a conversation with a system message.

  4. Send a user message asking "What are you?" to the assistant.

  5. Output the assistant's response to your console.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
